1. Identify specific data quality issues visible in the sample (do NOT speculate beyond the data).

- deduplication        – remove duplicate rows
- entity_resolution    – unify records that refer to the same real-world entity
- format_standardization – normalise dates, phones, postal codes, currency, etc.
- missing_value_imputation – fill null/empty values using statistical or semantic methods
- type_coercion        – fix columns whose dtype is wrong (e.g. number stored as string)

Rules:
- Be factual and concise.
- Only recommend tasks that are clearly warranted by the data you see.
- List recommended task names on separate lines starting with "TASK: ".
